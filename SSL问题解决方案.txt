# SSL证书问题解决方案

如果您在安装Python依赖时看到以下错误:

```
WARNING: Retrying (Retry(total=4, connect=None, read=None, redirect=None, status=None)) after connection broken by 'SSLError(SSLCertVerificationError(1, '[SSL: CERTIFICATE_VERIFY_FAILED] certificate verify failed: unable to get local issuer certificate (_ssl.c:1028)'))': /simple/tqdm/
```

这是由于您的系统无法验证PyPI服务器的SSL证书所致。以下是几种解决方法：

## 方法1: 使用提供的install_deps.bat

我们已经创建了一个特殊的批处理文件`install_deps.bat`，它会跳过SSL验证以安装依赖。
直接运行此文件或通过setup.bat安装依赖。

## 方法2: 手动配置pip跳过验证

如果您想手动安装，可以使用以下命令:

```
pip install --trusted-host pypi.org --trusted-host files.pythonhosted.org -r requirements.txt
```

## 方法3: 更新证书

您可以尝试更新您的Python的证书:

1. 下载最新的证书文件：https://curl.se/ca/cacert.pem
2. 将此文件保存为cacert.pem
3. 设置环境变量REQUESTS_CA_BUNDLE指向此文件
   - 在命令提示符中执行: `set REQUESTS_CA_BUNDLE=路径\到\cacert.pem`
   - 或在系统环境变量中永久设置此变量

## 方法4: 使用国内镜像源

您也可以使用国内的PyPI镜像，如清华镜像源:

```
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt
```

如果您仍然遇到问题，请参考Python和pip的官方文档获取更多关于SSL证书问题的解决方案。
