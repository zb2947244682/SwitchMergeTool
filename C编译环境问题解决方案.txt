# C++编译环境问题解决方案

在安装Python依赖时，您遇到了以下错误：

```
error: Microsoft Visual C++ 14.0 or greater is required. Get it with "Microsoft C++ Build Tools"
```

这是因为某些Python包（例如inflate64和pyppmd）需要编译C/C++代码，而您的系统上未安装必要的编译工具。

## 解决方案1：安装Microsoft Visual C++ Build Tools

1. 下载并安装Visual C++ Build Tools：
   - 访问 https://visualstudio.microsoft.com/visual-cpp-build-tools/
   - 下载并运行安装程序
   - 在安装选项中，选择"C++构建工具"选项
   - 完成安装后重新运行安装依赖项的命令

## 解决方案2：使用修改后的requirements.txt（当前已实施）

我们已经修改了requirements.txt文件，避免使用需要编译的包。现在直接运行安装命令应该可以正常工作。

## 解决方案3：使用预编译的wheel文件

如果您仍然遇到问题，可以下载预编译的wheel文件并手动安装：

1. 访问 https://www.lfd.uci.edu/~gohlke/pythonlibs/
2. 下载适合您Python版本的wheel文件
3. 使用pip install命令安装下载的wheel文件

例如：
```
pip install C:\path\to\downloaded\wheel\file.whl
```

## 注意事项

由于我们已经修改了依赖项列表，一些功能可能会受到限制。但基本的文件处理功能应该可以正常工作。如果您需要完整的功能，建议安装Visual C++ Build Tools。
